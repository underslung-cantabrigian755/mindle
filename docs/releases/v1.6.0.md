## 📝 Diff-on-reload

When something outside Mindle writes to the open file — vim, an agent, a sync daemon — Mindle now renders the change as a Word-style track-changes overlay you can review. Each chunk shows the deleted text struck through and the new text in soft green, with **✓ Keep** and **✗ Revert** chips inline.

- **Per-chunk control.** Accept or revert one paragraph at a time; the diff shrinks as you go.
- **Whole-document shortcuts.** `⌘⌥⏎` keeps everything; `⌘⌥⌫` reverts everything.
- **Reject writes through.** Clicking ✗ Revert on a chunk rewrites the file on disk back to the version you'd last reviewed — the agent that produced the change picks up your decision the next time it reads.
- **Survives restart.** An unfinished review is persisted in the sidecar, so closing and reopening the window picks up where you left off.

This is the foundation for the v2.0 MCP loop: the agent writes, you read, you mark up; the agent reads your annotations back via Mindle's MCP server (next release) and revises. Today it's already useful on its own — anyone editing the file in vim or any other editor outside Mindle gets visible change tracking.
